Constructors

Learning about Constructors at the KU Full Stack Bootcamp, Feb 2, 2019

Growing up in the aughts meant an unhealthy obsession with Beanie Babies (oh, yeah. I had over 200, including TWO Princess Diana’s), a forever battle between loving Backstreet Boys or N’Sync, the mandatory uniform of leggings with foot straps covered with white socks and an oversized sweatshirt, and an anxiety-filled check in with my Tamagotchi. So, you could imagine my surprise when I discovered that we would be making our own Tamagotchis with this handy tool called a Constructor.

Constructors are brilliant - I can set up an object with all of the pertinent details needed and simply add the details with each related variable. What’s more? I can add functions within my constructor to give my variables action or a purpose. If my Tamagotchi is tired? Create a function the gives the little guy sleepy time and restores his energy points to max. It is a fairly simple and fun tool to use, instead of creating separate objects for each variable. Some problems I ran in to: remembering to call a function with the variable name in front (i.e tamagotchi.feedMe()) and knowing when to create a function for an individual variable versus as a part of the constructor. Overall, it was a fun day of learning and nostalgia!
